## 段落

### 作品概要
大學四年級畢業製作。以「情緒影響天氣」為核心概念打造的日常養成戀愛遊戲，由 5 人團隊「兩個磨坊」歷時 **18 週**共同完成，入圍 **2026 放視大賞 PC 與主機遊戲組**（參賽編號 C26-GA-0255）。個人職務為主程式，負責整個遊戲框架從零開發，並參與 UX 玩法設計與劇情系統。

玩家扮演平凡的台北大學生，在某天邂逅了情緒能影響天氣的神祕少女「霏那」，並展開同居生活。在照顧霏那的日常、平衡台北含水值的同時，也逐漸加深彼此的羈絆。

### 核心玩法：情緒 × 天氣 × 數值養成
「含水值」（0 ~ 2000）是貫穿全局的核心數值，代表台北天氣的濕潤程度。每小時依當前天氣加減含水值（晴 −5、陰 ±0、雨 +5、颱風 +15），玩家必須透過讓霏那保持好情緒來控制天氣、穩定含水值。含水值歸零或爆表皆觸發對應 Bad End。

玩家以**體力**為唯一限制，可自由規劃每日行程——不受行動點數或回合制框架束縛，真實模擬日常生活節奏。霏那擁有 **14 種情緒狀態**，互相以狀態機轉換，情緒會同時影響養成數值的正負倍率與天氣失控機率。

### 日常互動小遊戲
遊戲嵌入多款輕量小遊戲，豐富每日的互動選項。各小遊戲以鬆耦合方式整合至主流程，各自獨立開發。

- **料理小遊戲**：食材搭配與火候控制，成果分 Great / Good / Bad，影響霏那心情
- **叫醒霏那 / 許願**：日常觸發的互動小遊戲，推動角色關係
- **過馬路（CrossyRoad 風格）**：外出時的 3D 過關挑戰，使用 RenderTexture 整合
- **商店街採購**：購買物品影響後續互動選項與劇情分支

### 程式架構
一人從零搭建整個遊戲框架，採用 Unity 經典 **Singleton + Coroutine + UnityAction Event-Bus** 架構，高內聚的 Manager 分工讓系統職責清晰。主要系統如下：

- **GameController**：總控；串接時間流逝、地點切換、情緒變化、結局觸發等所有跨系統演出 Coroutine
- **WeatherManager**：含水值演算 + 天氣自然變化定時器（4~12 / 24~72 小時隨機週期）
- **EventManager**：Dictionary 佇列劇情事件分派，支援條件鎖、失敗分支、下一階段註冊
- **DialogManager**：Code-driven Story + xNode 視覺化對話圖雙系統並行
- **SaveLoadManager**：JSON 多存檔槽 + 截圖縮圖 + AssemblyQualifiedName 反射還原事件佇列
- **EndingManager**：7+ 種結局的統一觸發與演出管理

技術規格：

- UI 效果：LeTai Translucent Image（毛玻璃）、TextMesh Pro
- 渲染：動態環境光 + 即時色彩偏移模組

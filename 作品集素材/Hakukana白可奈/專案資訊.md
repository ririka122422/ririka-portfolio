## 專案簡介

V-TYPE 是一個**場域導覽形象生成平台**，核心目標是讓任何場域（展覽館、校園、博物館等）都能快速建立一個專屬的 AI 虛擬導覽角色，透過 V-Type 形象結合 AI 技術，打造令使用者放心、親切的導覽服務體驗。

本專題以**國立台北科技大學**作為實作場域，產出了一名名叫「**白可奈**」的 AI 虛擬導覽員，並搭配 Unity 作為前端互動介面，實現文字對話、語音合成與多語言支援等功能。

### 設計理念

| 對象 | 價值 |
|------|------|
| **廠商／場域方** | 快速建立具有獨特個性的場域形象，提升展覽或活動的完整度與識別度 |
| **使用者** | 獲得更親切、更沉浸的導覽體驗，更快速地了解場域資訊 |

---

## 應用場合

平台適用於任何需要「讓使用者了解場域」的情境，主要流程如下：

```
發現使用者對場域不熟悉
        ↓
接洽平台，生成專屬虛擬導覽形象
        ↓
導入 V-Type 協助使用者了解場域
```

## 平台架構

平台分為**前台**與**後台**兩個主要組成：

### 前台（與使用者互動）

使用者透過 Unity App 與 V-Type 角色進行互動，支援：
- 文字輸入對話
- 語音輸入對話（規劃中）
- 環境識別 / AR 互動（規劃中）

互動流程如下：

```
使用者（語音或打字）
        ↓
    V-Type 前端（Unity）
        ↓            ↑
  後台資料庫  →  轉化回應  →  回應使用者
```

### 後台（與廠商對接）

負責接收廠商資料、生成角色形象與建立 AI 知識資料庫，最終產出可部署於前台的「場域虛擬形象」。

---

## 後台：形象產生流程

廠商透過後台平台，依照以下六步驟產出一個完整的 V-Type 場域形象：

```
接收需求 → AI(1)整理 → 廠商確認 → AI(2)生圖 → 廠商確認 → 交稿
               ↑____________↓            ↑____________↓
                  提出反饋                  提出反饋
```

### 步驟說明

**Step 1 — 廠商填寫資訊表單**  
廠商提供以下資料：官網、場域類型、主題特色介紹、場域亮點、地點與地圖資訊、特別活動、其他備註、期望角色特質等。

**Step 2 — AI(1) 整理角色資訊**  
平台以 AI 將廠商提供的原始資料整理成結構化的角色設定文件，包含主題概念、核心理念、目標情感（如「安心」）等，作為後續生圖的依據。

**Step 3 — 廠商確認並修正**  
廠商審閱 AI 整理的角色資訊，提出修改意見。例如：「要能展現『誠樸精勤』的精神」、「角色定位要更腳踏實地」。AI 根據反饋進行修正。

**Step 4 — AI(2) 生成角色圖像**  
依據確認後的角色設定，AI 生成多個不同個性方向的角色立繪，供廠商選擇。例如本次北科大案例產出了：
- 穩重踏實 / 實事求是 / 內斂冷靜
- 開朗外向 / 樂於嘗試 / 熱情好學
- **理性分析 / 冷靜沈著 / 注重細節** ← 廠商選定
- 活力十足 / 堅毅勇敢 / 團隊協作
- 樂觀友善 / 創新思維 / 適應力高

**Step 5 — 廠商確認並調整外觀**  
廠商對選定角色提出外觀調整意見。本次北科大反饋為「符合實務形象但略顯死板，希望更有生活感」，因此提供了五種穿搭風格選項：
- 學院宅系風（室內或秋冬）
- 可愛系街頭風（秋冬換季）
- 簡約文青風（自然輕熟）
- 溫暖休閒風（早秋初冬）
- **大學生戶外風（多功能實用）** ← 廠商選定

**Step 6 — 交稿**  
確認定稿，輸出至前台部署。

## 前台：AI 架構

前台系統由三個 AI 模型協同運作：

| 模型類型 | 功能 |
|---------|------|
| **大型語言模型（LLM）** | 理解使用者問題，查詢知識庫後決定回覆內容與角色行為動作 |
| **語音文字模型（TTS）** | 將 V-Type 的文字回應合成為具情緒感的語音輸出，同時支援使用者語音輸入轉文字 |
| **圖像生成模型（SD）** | 用於「跨次元合照」等趣味互動功能（規劃中） |

### AI 運作流程

```
使用者說話 / 使用者觸發互動（互動提示詞）
              ↓
           轉文字
              ↓
      決定回覆 ← 向量資料庫（場域知識）
         ↓          ↓
      轉語音      決定行為 ← 行為資料庫
         ↓          ↓
      說話聲音    畫面上表現
                （地圖指示 / 物品展示 / 跨次元合照 / 等等...）
```

## 前台：後端技術實作（白可奈系統）

此為本次專題的後端實作部分，以 Python Flask 搭建 AI 對話伺服器，並透過 RESTful API 與 Unity 前端串接。

### 技術棧

| 類別 | 技術 |
|------|------|
| 後端框架 | Python Flask + Flask-CORS |
| AI 代理框架 | LangChain + LangGraph（FSM 工作流） |
| 大型語言模型 | Google Gemini 2.5 Flash |
| 向量資料庫 | ChromaDB + dmeta-embedding-zh（Ollama） |
| 中英語音合成 | Coqui TTS — XTTS v2（支援音色複製） |
| 日文語音合成 | VOICEVOX Engine REST API |
| 硬體加速 | NVIDIA CUDA（GPU 推論） |
| 前端引擎 | Unity（C#） |

### 核心模組

```
agentServer.py   → Flask 主伺服器；LangGraph FSM 工作流主體
ai_prompts.py    → 所有 LangChain Prompt 模板管理
ai_tools.py      → RAG 查詢工具（北科知識 / 白可奈人設）
app.py           → XTTS v2 語音合成伺服器（Port 5001）
vocal_mod.py     → 語音生成排程；情緒語音參數對應
db_setting.py    → ChromaDB 初始化與資料載入
formate_funtion.py → JSON / dict / str 格式轉換工具
```

### LangGraph FSM 工作流

```
使用者輸入
    ↓
[start_node] — LLM 判斷：需要工具？還是直接回答？
    ↓                        ↓
[tool_node]            [style_node]
查向量資料庫           套用白可奈語氣 + 判斷情緒
    ↓                        ↓
回到 start_node         { answer, mood }
（帶查詢結果再判斷）        回傳 Unity 前端
```

### 向量資料庫設計

系統使用兩個獨立的 ChromaDB Collection，刻意分開以避免知識更新影響角色人設：

```
ntut_collection       → 北科校園知識（光華館、互動設計系...）
shirokana_collection  → 白可奈角色人設與對話風格
```

### 情緒驅動語音系統

根據 LLM 判斷的情緒，自動調整 VOICEVOX 的語音合成參數：

| 情緒 | 語速 | 音調 | 音量 | 抑揚頓挫 |
|------|------|------|------|---------|
| 開心 | 1.00 | +0.2 | 1.0 | 1.04 |
| 難過 | 0.92 | -0.04 | 0.8 | 0.08 |
| 害羞 | 0.95 | +0.06 | 0.7 | 0.09 |
| 生氣 | 1.08 | +0.03 | 1.1 | 1.06 |
| 驕傲 | 1.00 | 0.00 | 1.0 | 1.04 |

### API 端點

| 方法 | 路由 | 說明 |
|------|------|------|
| `POST` | `/api/chat` | 接收使用者訊息，回傳白可奈回答與情緒標籤 |
| `POST` | `/api/text_to_vocal` | 接收文字＋語言＋情緒，回傳 WAV 音檔（含自動翻譯） |
| `POST` | `/speak_raw` | XTTS v2 語音合成端點（Port 5001） |
| `GET` | `/api/health` | 健康檢查，回傳各資料庫筆數 |
| `GET` | `/api/test` | 測試端點 |

## 對話情境展示

### 情境一：問路
> 使用者 A 第一次來北科想參觀紅樓

```
使用者：我想問紅樓在哪裡

白可奈：從校門正門直走，看到超商後左轉就看到囉！
        旁邊有個大水池，需要為你帶路或給你地圖嗎？

使用者：好麻煩喔，你可以把我背過去嗎？

白可奈：包在我身上！（把你抬在身上）抓穩了嗎？我們出發！
        ……開玩笑的！我只是個虛擬導覽而已，還是動動你的雙腳吧。
```

### 情境二：情緒陪伴 + 資訊查詢
> 剛入學的使用者 B 向白可奈抱怨並詢問休息去處

```
使用者：今天入學第一天，聽了一個早上的導師廢話……

白可奈：欸～不可以這樣子說啦！導師們只是想讓你趕緊融入嘛！
        不過如果覺得很累，活動結束後趕緊去休息吧！

使用者：北科有什麼休息的好去處嗎？

白可奈：北科可以休息的地方很多喔！像是圖書館，裡面有沙發和
        座位區還有視聽室可以看電影喔！還有剛整修完的阿水廣場，
        是個可以欣賞戶外景色的地方！有想去的地方嗎？我可以帶路喔！
```

### 情境三：建築介紹 + AR 導覽
> 來賓使用者 C 經過紅樓好奇詢問，並開啟 AR 功能

```
使用者：剛才經過了一間看起來蠻古老的紅磚房屋，那是做什麼用的啊？

白可奈：你經過的是紅樓喔！是從以前工專時代留下的，現在展示著
        建築相關的歷史喔。感興趣的話可以去看看，說不定可以更了解北科呢！

使用者：原來如此，那是可以直接入場的嗎？

白可奈：可以的，參觀紅樓並不需要門票，也不用申請喔！從正門直接進入就可以了！

（使用者開啟 App AR 環境識別功能）

白可奈：這棟樓超級有歷史，最早是日治時期蓋的……
        原本是圖書館的附屬書庫，而且還是為了紀念日本皇室結婚25週年而建的，
        超奇特的用途對吧！
```
